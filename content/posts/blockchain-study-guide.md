블록체인을 처음 공부하려고 하면 거의 항상 용어에서 막힌다. EOA, nonce, gas, validator, smart contract, RPC, token standard 같은 단어가 한꺼번에 튀어나온다.

처음에는 용어 목록을 하나씩 외우면 될 것 같았다. 그런데 막상 정리해보면, 용어를 많이 외워도 "그래서 사용자가 버튼을 누르면 블록체인에서는 무슨 일이 일어나는가?"가 잡히지 않으면 계속 헷갈린다.

그래서 블록체인은 사전처럼 공부하기보다, 먼저 하나의 흐름을 따라가는 편이 좋다.

```text
지갑 -> 서명 -> 트랜잭션 -> 노드 -> 블록 -> 상태 변경
```

이 글은 블록체인 전체를 깊게 설명하는 글은 아니다. 처음 공부할 때 어떤 순서로 보면 덜 막히는지 정리해둔 학습 가이드에 가깝다.

## 먼저 전체 흐름만 보기

블록체인을 처음 볼 때 가장 먼저 잡을 문장은 이것이다.

```text
블록체인은 여러 노드가 같은 상태 변경 기록을 검증하고 공유하는 시스템이다.
```

여기서 중요한 단어는 "상태 변경"이다.

누군가 ETH를 보내면 잔액 상태가 바뀐다. 토큰을 전송하면 토큰 컨트랙트 안의 잔액 기록이 바뀐다. NFT를 민팅하면 소유자 기록이 바뀐다. DeFi에서 스왑을 하면 풀의 토큰 비율과 내 잔액이 바뀐다.

겉으로는 코인 전송, NFT 구매, 스왑, 예치처럼 다르게 보이지만 블록체인 입장에서 보면 대부분 이런 흐름이다.

1. 사용자가 지갑에서 어떤 행동을 선택한다.
2. 지갑이 트랜잭션 내용을 만든다.
3. 사용자가 개인키로 서명한다.
4. 트랜잭션이 네트워크에 전파된다.
5. 검증자가 트랜잭션을 블록에 포함한다.
6. 실행 결과에 따라 블록체인의 상태가 바뀐다.

처음에는 이 흐름만 반복해서 보는 편이 좋다. 세부 용어는 이 흐름 위에 하나씩 붙이면 된다.

## 1단계: 지갑을 계정으로 이해하기

처음에는 지갑을 "코인을 담는 앱"으로 생각하기 쉽다. 하지만 이더리움 계열에서 지갑은 더 정확히 말하면 계정을 다루는 인터페이스에 가깝다.

사용자가 흔히 쓰는 메타마스크 주소 같은 계정은 보통 EOA다. EOA는 개인키로 제어되는 계정이다. 개인키를 가진 사람이 트랜잭션에 서명할 수 있고, 그 서명이 있어야 네트워크가 "이 계정 주인이 승인한 행동이구나"라고 검증할 수 있다.

이 단계에서 먼저 볼 용어는 많지 않다.

- 지갑
- 주소
- 개인키
- 공개키
- 시드 구문
- EOA
- 서명

여기서 가장 중요한 감각은 "블록체인 자산은 지갑 앱 안에 들어 있는 파일이 아니다"라는 점이다. 실제 기록은 블록체인 네트워크의 상태에 있고, 지갑은 개인키로 그 상태 변경을 요청하는 도구에 가깝다.

그래서 처음 실습할 때는 반드시 테스트 전용 지갑을 따로 만드는 편이 좋다. 실제 자산이 들어 있는 지갑, 평소 쓰는 시드 구문, 회사나 개인의 중요한 계정은 학습 실습에 쓰지 않는 것이 안전하다.

## 2단계: 트랜잭션 하나를 끝까지 따라가기

지갑 다음에는 트랜잭션을 봐야 한다. 블록체인을 이해하는 가장 좋은 단위는 "체인 전체"가 아니라 트랜잭션 하나다.

예를 들어 테스트넷에서 아주 작은 ETH를 다른 테스트 주소로 보냈다고 생각해보자. 그 다음에는 블록 탐색기에서 트랜잭션 해시를 열어본다.

처음에는 아래 항목만 보면 된다.

- `Status`: 성공했는지 실패했는지
- `From`: 누가 보냈는지
- `To`: 누구에게 보냈는지
- `Value`: 얼마를 보냈는지
- `Transaction Fee`: 수수료가 얼마나 들었는지
- `Block`: 어떤 블록에 포함됐는지
- `Nonce`: 해당 계정의 몇 번째 트랜잭션인지

이걸 한 번 보면 용어가 갑자기 조금 현실적으로 바뀐다.

`from`은 내 지갑 주소다. `to`는 받는 주소다. `value`는 전송한 금액이다. `fee`는 네트워크에 지불한 비용이다. `block`은 내 요청이 어느 기록 묶음에 들어갔는지 보여준다.

처음부터 합의 알고리즘을 깊게 파기보다, 내가 보낸 트랜잭션 하나가 어떤 필드로 구성되는지 보는 편이 훨씬 빠르다.

## 3단계: 가스는 수수료가 아니라 실행 비용으로 보기

가스는 처음에 가장 헷갈리는 단어 중 하나다. 그냥 "수수료"라고 외우면 어느 정도는 맞지만, 스마트 컨트랙트까지 들어가면 부족하다.

가스는 블록체인에서 연산과 저장 공간을 쓰는 비용을 재는 단위에 가깝다.

단순 ETH 전송은 비교적 단순한 상태 변경이다. 반면 스마트 컨트랙트 함수를 실행하면 내부 코드가 돌고, 저장소를 읽거나 쓰고, 이벤트를 남길 수 있다. 당연히 더 많은 계산 자원이 필요할 수 있다.

그래서 이 단계에서는 이렇게 구분해서 보면 좋다.

- ETH 전송: 계정 잔액을 바꾸는 트랜잭션
- 컨트랙트 호출: 특정 주소의 코드 실행을 요청하는 트랜잭션
- 가스 사용량: 그 실행에 실제로 든 계산량
- 가스비: 사용량에 가스 가격을 곱해 지불한 비용

처음부터 `base fee`, `priority fee`, `max fee`를 모두 외우려고 하면 무겁다. 먼저 "트랜잭션은 공짜가 아니고, 실행이 복잡할수록 비용이 생긴다"는 감각을 잡는 것으로 충분하다.

## 4단계: 스마트 컨트랙트는 주소를 가진 프로그램으로 보기

스마트 컨트랙트는 이름 때문에 처음에는 법적 계약서처럼 느껴질 수 있다. 하지만 개발자 관점에서는 블록체인 위에 배포된 프로그램에 가깝다.

조금 더 풀면 이런 느낌이다.

```text
스마트 컨트랙트 = 주소를 가지고 있고, 상태를 저장할 수 있고, 트랜잭션으로 함수를 실행할 수 있는 프로그램
```

일반 EOA 주소는 개인키가 제어한다. 반면 컨트랙트 계정은 코드가 제어한다. 사용자가 컨트랙트 주소로 트랜잭션을 보내면, 그 컨트랙트 안의 함수가 실행되고 결과에 따라 상태가 바뀐다.

이 단계에서는 아주 작은 예시만 보면 된다.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract Counter {
    uint256 public count;

    function increase() public {
        count += 1;
    }
}
```

이 코드를 깊게 분석할 필요는 없다. 처음에는 세 가지만 보면 된다.

- `count`는 컨트랙트가 들고 있는 상태다.
- `increase()`는 상태를 바꾸는 함수다.
- 누군가 `increase()`를 트랜잭션으로 호출하면 `count`가 바뀐다.

즉 스마트 컨트랙트 학습의 출발점도 결국 상태 변경이다.

## 5단계: 토큰은 별도 코인이 아니라 컨트랙트의 기록으로 보기

토큰에서 한 번 더 헷갈린다. ETH 같은 네이티브 코인과 ERC-20 토큰을 같은 방식으로 생각하면 중간에 막힌다.

ETH 잔액은 이더리움 계정의 기본 잔액이다. 반면 ERC-20 토큰 잔액은 보통 토큰 컨트랙트 안의 기록이다.

아주 단순하게 보면 ERC-20 컨트랙트 안에는 이런 정보가 있다고 생각할 수 있다.

```text
주소 A -> 100개
주소 B -> 50개
주소 C -> 0개
```

토큰을 전송한다는 것은 내 지갑에서 파일을 보내는 일이 아니다. 토큰 컨트랙트의 `transfer` 같은 함수를 호출해서 "A의 잔액을 줄이고 B의 잔액을 늘리는 상태 변경"을 실행하는 일에 가깝다.

이 관점이 잡히면 `approval`도 조금 덜 낯설다.

`approval`은 내 토큰을 누군가에게 바로 보내는 행동이 아니다. 특정 컨트랙트가 내 토큰을 정해진 범위 안에서 대신 사용할 수 있도록 허용하는 상태 변경이다. 그래서 DeFi를 공부할 때는 `approve`와 `swap`을 반드시 나눠서 봐야 한다.

## 6단계: 작은 실습 하나를 정해서 반복하기

블록체인은 읽기만 하면 금방 추상적으로 변한다. 그래서 처음에는 큰 프로젝트보다 작은 실습 하나를 반복하는 편이 좋다.

추천하는 첫 실습은 이 정도다.

1. 테스트 전용 지갑을 만든다.
2. 테스트넷 ETH를 받는다.
3. 다른 테스트 주소로 소량을 보낸다.
4. 트랜잭션 해시를 블록 탐색기에서 확인한다.
5. 같은 주소로 토큰 컨트랙트를 하나 호출해본다.
6. 실패한 트랜잭션도 일부러 하나 만들어보고 에러를 읽어본다.

여기서 목표는 돈을 보내는 법을 배우는 게 아니다. 내가 만든 행동이 블록체인에서 어떤 기록으로 남는지 보는 것이다.

가능하면 매번 아래 표를 직접 채워보면 좋다.

| 항목 | 확인할 것 |
| --- | --- |
| From | 어떤 계정이 서명했는가 |
| To | EOA인가, 컨트랙트인가 |
| Value | 네이티브 코인이 이동했는가 |
| Input Data | 컨트랙트 함수 호출이 있는가 |
| Gas | 실행 비용이 얼마나 들었는가 |
| Logs | 어떤 이벤트가 남았는가 |

이 표를 몇 번 채우면 탐색기가 더 이상 낯선 화면처럼 보이지 않는다.

## 7단계: 보안은 맨 마지막이 아니라 처음부터 보기

블록체인 학습에서 보안은 고급 주제처럼 보이지만, 사실은 가장 처음부터 같이 봐야 한다.

처음에는 스마트 컨트랙트 취약점보다 사용자 보안이 더 중요하다.

- 시드 구문은 절대 공유하지 않는다.
- 테스트용 지갑과 실제 지갑을 분리한다.
- 모르는 사이트에서 서명 요청을 함부로 승인하지 않는다.
- `approve`는 "로그인"이 아니라 권한 부여일 수 있다.
- 트랜잭션의 `to`, `value`, 네트워크를 확인하는 습관을 들인다.

개발 쪽으로 들어간 뒤에는 컨트랙트 보안을 따로 봐야 한다. 재진입 공격, 접근 제어, 정수 처리, 업그레이드 가능한 컨트랙트, 오라클 의존성 같은 주제가 나온다.

하지만 처음부터 모든 공격 기법을 외울 필요는 없다. 먼저 "서명은 곧 승인이고, 승인된 트랜잭션은 되돌리기 어렵다"는 감각을 잡는 것이 중요하다.

## 4주 학습 루트

처음 공부한다면 한 번에 모든 주제를 보려고 하지 않는 편이 좋다. 아래처럼 4주 정도로 나눠보면 부담이 줄어든다.

### 1주차: 계정과 트랜잭션

목표는 지갑으로 만든 행동이 트랜잭션으로 남는다는 것을 이해하는 것이다.

- 지갑, 주소, 개인키, 공개키를 구분한다.
- EOA와 컨트랙트 계정을 구분한다.
- 테스트넷에서 트랜잭션 하나를 보내본다.
- 블록 탐색기에서 `from`, `to`, `value`, `fee`, `nonce`를 확인한다.

### 2주차: 블록과 가스

목표는 트랜잭션이 네트워크에서 처리되는 방식을 이해하는 것이다.

- 트랜잭션이 블록에 포함된다는 의미를 본다.
- pending, success, failed 상태를 구분한다.
- 가스가 왜 필요한지 이해한다.
- 같은 전송이라도 네트워크 상황에 따라 수수료가 달라질 수 있음을 확인한다.

### 3주차: 스마트 컨트랙트

목표는 컨트랙트를 "주소를 가진 프로그램"으로 보는 것이다.

- 아주 작은 Counter 컨트랙트를 읽어본다.
- 상태 변수와 함수를 구분한다.
- 읽기 호출과 쓰기 트랜잭션을 구분한다.
- 컨트랙트 호출 결과가 탐색기에 어떻게 남는지 본다.

### 4주차: 토큰과 dApp 흐름

목표는 지갑, 컨트랙트, 프론트엔드가 어떻게 이어지는지 보는 것이다.

- ERC-20 토큰 전송 흐름을 본다.
- `approve`와 실제 실행 트랜잭션을 나눠서 본다.
- dApp에서 지갑 연결과 서명 요청이 언제 발생하는지 확인한다.
- 내가 누른 버튼이 어떤 컨트랙트 호출로 이어지는지 추적한다.

## 처음에 보지 않아도 되는 것들

처음부터 너무 깊게 들어가지 않아도 되는 주제도 있다.

- 모든 합의 알고리즘의 세부 수식
- L2별 내부 구조 비교
- MEV의 세부 전략
- 브릿지 보안 모델
- 복잡한 DeFi 수익률 전략
- 컨트랙트 업그레이드 패턴 전체

이 주제들이 중요하지 않다는 뜻은 아니다. 다만 처음에는 이것들이 기본 흐름을 가리는 경우가 많다. 지갑, 서명, 트랜잭션, 블록, 상태 변경이 먼저다.

## 정리

블록체인을 처음 공부할 때는 용어를 많이 외우는 것보다 하나의 트랜잭션을 끝까지 따라가는 편이 낫다.

```text
누가 서명했는가?
무엇을 요청했는가?
어떤 주소로 보냈는가?
실행 비용은 얼마였는가?
결과적으로 어떤 상태가 바뀌었는가?
```

이 다섯 질문에 답할 수 있으면, EOA, gas, smart contract, token, approval 같은 용어가 따로 흩어지지 않는다. 모두 같은 흐름 안에서 자기 자리를 갖게 된다.

처음 목표는 블록체인을 전부 아는 것이 아니라, 블록체인에서 일어나는 일을 읽을 수 있게 되는 것이다. 그 다음부터는 토큰, DeFi, NFT, L2, 컨트랙트 개발 같은 주제로 천천히 넓혀가면 된다.
