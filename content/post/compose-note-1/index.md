# 내가 생각하는 컴포즈의 장점

특정 기술의 장점을 얘기하려면 사실 많이 사용해본뒤에 얘기할 수 있는 부분이 아닌가 싶다.

일단 컴포즈를 많이 사용하지 않은 상태에서 느낀 컴포즈의 장점은 

- 이전 안드로이드 뷰 시스템의 경우 최상위 View 객체를 계속 상속받아서 만들어진 구조라 
  
  변경이 어렵고 구조 파악이 힘들었는데 컴포즈의 경우 컴포저블 함수를 조합해서 UI 를 만들기 때문에 유지보수가 쉽다.

- 안드로이드 뷰시스템의 경우 명령형 방식이라 직접 if 라던가 이런 코드들로 직접 UI 를 조작했었는데

  컴포즈는 선언적 방식이라 UI 상태, 구성요소를 명시적으로 정의하고. 

  실제 UI 업데이트는 컴포즈 시스템에서 처라가 되므로 코드 복잡도가 많이 줄었다.
  
  아직까지는 이부분이 가장 큰 장점으로 느껴진다.


- 데이터바인딩 개념 자체는 훌륭하지만 XML 에서 적용하기에는 복잡한 수식이라던가 문법들이 가독성이 좋지않다.

- 프리뷰는 글쎄..아직 버그가 좀 있어서 차후 업데이트가 되면 편리할 것 같다.
