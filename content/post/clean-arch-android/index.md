# 안드로이드에서의 클린아키텍처


아키텍처에 대한 나의 생각:

- 아키텍처의 필요성은 앱의 규모와 참여 개발자 수(능력)에 따라 다르다.
    + 앱 프로젝트 크기가 크거나 개발자 수가 많을 수록 더 필요해진다.
- 아키텍처의 세분화와 코드 복잡성은 비례한다.
    + 아키텍처는 기본적으로 역할을 구분하는데 사용된다.
    + 아키텍처가 복잡해질수록 클래스와 함수는 많아지면 이에 따른 러닝커브가 증가한다.  
- 반드시 옳은 아키텍처는 없다. (매우 중요)
    + MVC, MVP, MVVM 각각의 구조적인 특징을 구분해서 사용해야한다.
    + 그럼에도 화면(UI) 와 비즈니스 로직은 구분되어야한다.
